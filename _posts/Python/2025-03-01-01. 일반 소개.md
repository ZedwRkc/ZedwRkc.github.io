이 책의 첫 번째 목표는 파이썬으로 프로그래밍하는 방법을 가르치는 것입니다.

하지만 프로그래밍을 배운다는 것은 새로운 사고방식을 배우는 것을 의미하므로, 이 책의 두 번째 목표는 여러분이 컴퓨터 과학자처럼 생각하도록 돕는 것입니다. 이러한 사고방식은 수학, 공학, 자연 과학의 가장 좋은 특징들을 결합합니다. 수학자처럼 컴퓨터 과학자들은 아이디어, 특히 계산을 나타내기 위해 형식 언어를 사용합니다. 엔지니어처럼 그들은 사물을 설계하고, 구성 요소를 시스템으로 조립하며 대안들 사이의 장단점을 평가합니다. 과학자처럼 그들은 복잡한 시스템의 행동을 관찰하고, 가설을 세우고, 예측을 테스트합니다.

우리는 프로그래밍의 가장 기본적인 요소부터 시작하여 점차적으로 나아갈 것입니다. 이 장에서는 파이썬이 숫자, 문자, 단어를 어떻게 나타내는지 알아볼 것입니다. 그리고 산술 연산을 수행하는 방법을 배우게 됩니다.

또한 `연산자`, `표현식`, `값`, `타입`과 같은 프로그래밍 어휘를 배우기 시작할 것입니다. 이 어휘는 책의 나머지 부분을 이해하고, 다른 프로그래머와 소통하며, 가상 비서를 사용하고 이해하는 데 필요하기 때문에 중요합니다.

## **1.1. 산술 연산자**

산술 연산자는 산술 계산을 나타내는 기호입니다. 예를 들어, 더하기 기호 `+`는 덧셈을 수행합니다.

```python
38 + 12
````

```
42
```

빼기 기호 `-`는 뺄셈을 수행하는 연산자입니다.

```python
43 - 1
```

```
42
```

별표 `*`는 곱셈을 수행합니다.

```python
6 * 7
```

```
42
```

그리고 슬래시 `/`는 나눗셈을 수행합니다:

```python
84 / 2
```

```
42.0
```

나눗셈의 결과가 `42`가 아닌 `42.0`이라는 점에 주목하세요. 이는 파이썬에 두 가지 타입의 숫자가 있기 때문입니다:

  * \*\*정수(integers)\*\*는 분수나 소수 부분이 없는 숫자를 나타냅니다.
  * \*\*부동소수점 숫자(floating-point numbers)\*\*는 정수와 소수점이 있는 숫자를 나타냅니다.

두 정수를 더하거나, 빼거나, 곱하면 결과는 정수입니다. 하지만 두 정수를 나누면 결과는 부동소수점 숫자입니다. 파이썬은 정수 나눗셈을 수행하는 또 다른 연산자 `//`를 제공합니다. 정수 나눗셈의 결과는 항상 정수입니다.

```python
84 // 2
```

```
42
```

정수 나눗셈은 항상 내림(바닥 쪽으로)하기 때문에 "버림 나눗셈(floor division)"이라고도 불립니다.

```python
85 // 2
```

```
42
```

마지막으로, 연산자 `**`는 거듭제곱을 수행합니다. 즉, 숫자를 거듭제곱합니다:

```python
7 ** 2
```

```
49
```

다른 언어에서는 캐럿 `^`이 거듭제곱에 사용되지만, 파이썬에서는 XOR이라는 비트 연산자입니다. 비트 연산자에 익숙하지 않다면 결과가 예상과 다를 수 있습니다:

```python
7 ^ 2
```

```
5
```

이 책에서는 비트 연산자를 다루지 않지만, http://wiki.python.org/moin/BitwiseOperators 에서 읽어볼 수 있습니다.

## **1.2. 표현식**

연산자와 숫자의 조합을 \*\*표현식(expression)\*\*이라고 합니다. 표현식은 여러 개의 연산자와 숫자를 포함할 수 있습니다. 예를 들어, 여기 두 개의 연산자를 포함하는 표현식이 있습니다.

```python
6 + 6 ** 2
```

```
42
```

덧셈보다 거듭제곱이 먼저 수행된다는 점에 주목하세요. 파이썬은 수학 수업에서 배웠을 연산 순서를 따릅니다: 거듭제곱이 곱셈과 나눗셈보다 먼저 수행되고, 곱셈과 나눗셈은 덧셈과 뺄셈보다 먼저 수행됩니다.

다음 예제에서는 덧셈보다 곱셈이 먼저 수행됩니다.

```python
12 + 5 * 6
```

```
42
```

덧셈을 먼저 수행하고 싶다면 괄호를 사용할 수 있습니다.

```python
(12 + 5) * 6
```

```
102
```

모든 표현식은 \*\*값(value)\*\*을 가집니다. 예를 들어, 표현식 `6 * 7`은 값 `42`를 가집니다.

## **1.3. 산술 함수**

산술 연산자 외에도 파이썬은 숫자와 함께 작동하는 몇 가지 함수를 제공합니다.

예를 들어, `round` 함수는 부동소수점 숫자를 받아 가장 가까운 정수로 반올림합니다.

```python
round(42.4)
```

```
42
```

```python
round(42.6)
```

```
43
```

`abs` 함수는 숫자의 절댓값을 계산합니다. 양수의 경우 절댓값은 숫자 자체입니다.

```python
abs(42)
```

```
42
```

음수의 경우 절댓값은 양수입니다.

```python
abs(-42)
```

```
42
```

이와 같이 함수를 사용할 때 우리는 함수를 \*\*호출(calling)\*\*한다고 말합니다. 함수를 호출하는 표현식을 \*\*함수 호출(function call)\*\*이라고 합니다. 

함수를 호출할 때는 괄호가 필수입니다. 괄호를 생략하면 오류 메시지가 나타납니다. 

```python
abs 42
```

```
  Cell In[17], line 1
    abs 42
    ^
SyntaxError: invalid syntax
```

이 메시지의 첫 줄은 지금 당장 이해할 필요가 없는 정보이므로 무시해도 됩니다. 두 번째 줄은 오류가 포함된 코드이며, 오류가 발견된 위치를 나타내기 위해 그 아래에 캐럿(`^`)이 있습니다. 마지막 줄은 이것이 \*\*구문 오류(SyntaxError)\*\*임을 나타내며, 이는 표현식의 구조에 문제가 있다는 것을 의미합니다. 이 예제에서 문제는 함수 호출에 괄호가 필요하다는 것입니다.

괄호와 값을 모두 생략하면 어떻게 되는지 봅시다.

```python
abs
```

```
<function abs(x, /)>
```

함수 이름 자체는 값을 가지는 합법적인 표현식입니다. 값이 표시될 때, 이는 `abs`가 함수임을 나타내며, 나중에 설명할 추가 정보를 포함합니다.

## **1.4. 문자열**

숫자 외에도 파이썬은 문자의 순서를 나타낼 수 있는데, 이를 \*\*문자열(strings)\*\*이라고 합니다. 문자들이 목걸이의 구슬처럼 함께 꿰어져 있기 때문입니다. 문자열을 작성하려면 문자 순서를 작은따옴표 안에 넣으면 됩니다.

```python
'Hello'
```

```
'Hello'
```

큰따옴표를 사용하는 것도 합법적입니다.

```python
"world"
```

```
'world'
```

큰따옴표를 사용하면 작은따옴표와 같은 기호인 아포스트로피를 포함하는 문자열을 쉽게 작성할 수 있습니다.

```python
"it's a small"
```

```
"it's a small"
```

문자열에는 공백, 구두점, 숫자도 포함될 수 있습니다.

`+` 연산자는 문자열과 함께 작동합니다. 두 문자열을 하나의 문자열로 결합하는데, 이를 \*\*이어붙이기(concatenation)\*\*라고 합니다.

```python
'Well, ' + "it's a small " + 'world.'
```

```
"Well, it's a small world."
```

`*` 연산자도 문자열과 함께 작동합니다. 문자열의 여러 복사본을 만들어 이어붙입니다.

```python
'Spam, ' * 4
```

```
'Spam, Spam, Spam, Spam, '
```

다른 산술 연산자들은 문자열과 함께 작동하지 않습니다.

파이썬은 문자열의 길이를 계산하는 `len`이라는 함수를 제공합니다.

```python
len('Spam')
```

```
4
```

`len`은 따옴표 사이의 문자는 세지만 따옴표 자체는 세지 않는다는 점에 유의하세요.

## **1.5. 값과 타입**

지금까지 우리는 세 가지 종류의 값을 보았습니다:
`2`는 정수입니다.
`42.0`은 부동소수점 숫자입니다.
`'Hello'`는 문자열입니다.

값의 종류를 \*\*타입(type)\*\*이라고 합니다. 모든 값은 타입을 가지거나, 때로는 특정 타입에 "속한다"고 말합니다.

파이썬은 어떤 값의 타입이든 알려주는 `type`이라는 함수를 제공합니다. 정수의 타입은 `int`입니다.

```python
type(2)
```

```
int
```

부동소수점 숫자의 타입은 `float`입니다.

```python
type(42.0)
```

```
float
```

그리고 문자열의 타입은 `str`입니다.

```python
type('Hello, World!')
```

```
str
```

타입 `int`, `float`, `str`은 함수처럼 사용될 수 있습니다. 예를 들어, `int`는 부동소수점 숫자를 받아 정수로 변환할 수 있습니다(항상 내림).

```python
int(42.9)
```

```
42
```

그리고 `float`는 정수를 부동소수점 값으로 변환할 수 있습니다.

```python
float(42)
```

```
42.0
```

이제 혼란스러울 수 있는 것이 있습니다. 숫자 시퀀스를 따옴표 안에 넣으면 어떻게 될까요?

```python
'126'
```

```
'126'
```

숫자처럼 보이지만 실제로는 문자열입니다.

```python
type('126')
```

```
str
```

이것을 숫자처럼 사용하려고 하면 오류가 발생할 수 있습니다.

```python
'126' / 3
```

```
TypeError: unsupported operand type(s) for /: 'str' and 'int'
```

이 예제는 `TypeError`를 생성하는데, 이는 표현식의 값들, 즉 \*\*피연산자(operands)\*\*들이 잘못된 타입을 가지고 있음을 의미합니다. 오류 메시지는 `/` 연산자가 이 값들의 타입, 즉 `str`과 `int`를 지원하지 않음을 나타냅니다.

숫자를 포함하는 문자열이 있다면 `int`를 사용하여 정수로 변환할 수 있습니다.

```python
int('126') / 3
```

```
42.0
```

숫자와 소수점을 포함하는 문자열이 있다면 `float`를 사용하여 부동소수점 숫자로 변환할 수 있습니다.

```python
float('12.6')
```

```
12.6
```

큰 정수를 쓸 때 `1,000,000`처럼 숫자 그룹 사이에 쉼표를 사용하고 싶을 수 있습니다. 이것은 파이썬에서 합법적인 표현식이지만, 결과는 정수가 아닙니다.

```python
1,000,000
```

```
(1, 0, 0)
```

파이썬은 `1,000,000`을 쉼표로 구분된 정수 시퀀스로 해석합니다. 이런 종류의 시퀀스에 대해서는 나중에 더 배울 것입니다.

밑줄을 사용하여 큰 숫자를 더 쉽게 읽을 수 있도록 할 수 있습니다.

```python
1_000_000
```

```
1000000
```

## **1.6. 형식 언어와 자연어**

\*\*자연어(Natural languages)\*\*는 사람들이 말하는 언어, 예를 들어 영어, 스페인어, 프랑스어 등입니다. 이것들은 사람들에 의해 설계된 것이 아니라 자연스럽게 진화했습니다.

\*\*형식 언어(Formal languages)\*\*는 특정 응용 프로그램을 위해 사람들이 설계한 언어입니다. 예를 들어, 수학자들이 사용하는 표기법은 숫자와 기호 간의 관계를 나타내는 데 특히 좋은 형식 언어입니다. 마찬가지로, 프로그래밍 언어는 계산을 표현하기 위해 설계된 형식 언어입니다.

형식 언어와 자연어는 몇 가지 공통된 특징을 가지고 있지만, 중요한 차이점들이 있습니다:

  * **모호성(Ambiguity)**: 자연어는 모호함으로 가득 차 있으며, 사람들은 문맥적 단서와 다른 정보를 사용하여 이를 처리합니다. 형식 언어는 거의 또는 완전히 모호하지 않도록 설계되었으며, 이는 어떤 프로그램이든 문맥에 관계없이 정확히 하나의 의미를 가진다는 것을 의미합니다.
  * **중복성(Redundancy)**: 모호함을 보완하고 오해를 줄이기 위해 자연어는 중복성을 사용합니다. 결과적으로, 종종 장황합니다. 형식 언어는 덜 중복적이고 더 간결합니다.
  * **축어성(Literalness)**: 자연어는 관용구와 은유로 가득합니다. 형식 언어는 말하는 그대로를 의미합니다.

우리 모두 자연어를 말하며 자라기 때문에 형식 언어에 적응하기가 때로는 어렵습니다. 형식 언어는 자연어보다 더 밀도가 높기 때문에 읽는 데 시간이 더 오래 걸립니다. 또한, 구조가 중요하기 때문에 항상 위에서 아래로, 왼쪽에서 오른쪽으로 읽는 것이 최선은 아닙니다. 마지막으로, 세부 사항이 중요합니다. 자연어에서는 넘어갈 수 있는 철자 및 구두점의 작은 오류가 형식 언어에서는 큰 차이를 만들 수 있습니다.

## **1.7. 디버깅**

프로그래머는 실수를 합니다. 기발한 이유로 프로그래밍 오류는 \*\*버그(bugs)\*\*라고 불리고, 그것들을 찾아내는 과정을 \*\*디버깅(debugging)\*\*이라고 합니다.

프로그래밍, 특히 디버깅은 때때로 강한 감정을 불러일으킵니다. 어려운 버그와 씨름하고 있다면 화가 나거나, 슬프거나, 당황스러울 수 있습니다. 이러한 반응에 대비하면 대처하는 데 도움이 될 수 있습니다. 한 가지 접근 방식은 컴퓨터를 속도와 정밀성과 같은 특정 강점과 공감 부족, 큰 그림을 파악하지 못하는 것과 같은 특정 약점을 가진 직원으로 생각하는 것입니다.

여러분의 임무는 좋은 관리자가 되는 것입니다: 강점을 활용하고 약점을 완화할 방법을 찾으세요. 그리고 감정을 사용하여 문제에 참여하되, 반응이 효과적으로 일하는 능력을 방해하지 않도록 하는 방법을 찾으세요.

디버깅을 배우는 것은 좌절스러울 수 있지만, 프로그래밍 이외의 많은 활동에 유용한 귀중한 기술입니다. 각 장의 끝에는 이와 같은 디버깅에 대한 제안 섹션이 있습니다. 도움이 되기를 바랍니다\!

## **1.8. 용어 사전**

  * **산술 연산자(arithmetic operator)**: `+`, `*`와 같이 덧셈이나 곱셈 같은 산술 연산을 나타내는 기호.
  * **정수(integer)**: 분수나 소수 부분이 없는 숫자를 나타내는 타입.
  * **부동소수점(floating-point)**: 정수와 소수 부분이 있는 숫자를 나타내는 타입.
  * **정수 나눗셈(integer division)**: 두 숫자를 나누고 정수로 내림하는 연산자 `//`.
  * **표현식(expression)**: 변수, 값, 연산자의 조합.
  * **값(value)**: 정수, 부동소수점 숫자, 문자열 또는 나중에 보게 될 다른 종류의 값 중 하나.
  * **함수(function)**: 유용한 연산을 수행하는 이름 있는 문장들의 시퀀스. 함수는 인수를 받을 수도 있고 받지 않을 수도 있으며, 결과를 생성할 수도 있고 생성하지 않을 수도 있습니다.
  * **함수 호출(function call)**: 함수를 실행하는 표현식 또는 표현식의 일부. 함수 이름 뒤에 괄호 안의 인수 목록으로 구성됩니다.
  * **구문 오류(syntax error)**: 프로그램을 파싱할 수 없게 만들어 실행을 불가능하게 하는 오류.
  * **문자열(string)**: 문자의 시퀀스를 나타내는 타입.
  * **이어붙이기(concatenation)**: 두 문자열을 끝과 끝으로 결합하는 것.
  * **타입(type)**: 값의 범주. 지금까지 본 타입은 정수(`int`), 부동소수점 숫자(`float`), 문자열(`str`)입니다.
  * **피연산자(operand)**: 연산자가 연산을 수행하는 값 중 하나.
  * **자연어(natural language)**: 사람들이 말하는, 자연스럽게 진화한 모든 언어.
  * **형식 언어(formal language)**: 수학적 아이디어나 컴퓨터 프로그램을 나타내는 것과 같은 특정 목적을 위해 사람들이 설계한 모든 언어. 모든 프로그래밍 언어는 형식 언어입니다.
  * **버그(bug)**: 프로그램의 오류.
  * **디버깅(debugging)**: 오류를 찾아 수정하는 과정.

#### **1.9.2. 연습문제 풀이**

숫자가 0.5로 끝날 때 `round` 함수가 어떻게 동작하는지 궁금할 수 있습니다. 정답은 때로는 올림하고 때로는 내림한다는 것입니다. 다음 예제를 시도해보고 어떤 규칙을 따르는지 알아보세요.

  * **`round(42.5)`**
    ```python
    print(round(42.5))
    ```
    ```
    42
    ```
  * **`round(43.5)`**
    ```python
    print(round(43.5))
    ```
    ```
    44
    ```

**정답:**
파이썬 3에서는 `.5`로 끝나는 숫자를 반올림할 때 **가장 가까운 짝수**로 반올림하는 "Round Half to Even" 방식을 사용합니다.

  * `round(42.5)`의 경우, 가장 가까운 짝수는 42이므로 결과는 `42`입니다.
  * `round(43.5)`의 경우, 가장 가까운 짝수는 44이므로 결과는 `44`입니다.

#### **1.9.3. 연습문제 풀이**

새로운 기능을 배울 때는 직접 시도해보고 일부러 실수를 만들어봐야 합니다. 그렇게 함으로써 오류 메시지를 배우게 되고, 나중에 다시 보았을 때 그 의미를 알 수 있습니다.

1.  **마이너스 기호를 사용하여 `-2`와 같은 음수를 만들 수 있습니다. 숫자 앞에 플러스 기호를 붙이면 어떻게 되나요? `2++2`는 어떤가요?**

    **정답:**

      * 숫자 앞에 플러스 기호(`+`)를 붙여도 값은 변하지 않습니다. `+2`는 그냥 `2`입니다.
      * `2++2`는 `2 + (+2)`로 해석되어 결과는 `4`가 됩니다.

2.  **연산자 없이 `4 2`처럼 두 개의 값을 나열하면 어떻게 되나요?**

    **정답:**
    `SyntaxError: invalid syntax` (잘못된 구문) 오류가 발생합니다. 파이썬은 두 값 사이에 어떤 연산을 수행해야 하는지 알 수 없기 때문입니다.

3.  **`round(42.5)`와 같은 함수를 호출할 때 괄호 하나 또는 둘 다를 생략하면 어떻게 되나요?**

    **정답:**
    `SyntaxError`가 발생합니다. 함수를 호출하려면 반드시 함수 이름 뒤에 괄호 `()`가 와야 합니다.

#### **1.9.4. 연습문제 풀이**

모든 표현식은 값을 가지고, 모든 값은 타입을 가지며, `type` 함수를 사용하여 어떤 값의 타입을 찾을 수 있다는 것을 기억하세요. 다음 표현식의 값의 타입은 무엇일까요?

**정답:**

  * `765` → **int** (정수)
  * `2.718` → **float** (부동소수점)
  * `'2 pi'` → **str** (문자열)
  * `abs(-7)` → `abs(-7)`의 값은 `7`이므로, 타입은 **int** (정수)입니다.
  * `abs(-7.0)` → `abs(-7.0)`의 값은 `7.0`이므로, 타입은 **float** (부동소수점)입니다.
  * `abs` → **builtin\_function\_or\_method** (내장 함수 또는 메소드)
  * `int` → **type** (타입)
  * `type` → **type** (타입)

#### **1.9.5. 연습문제 풀이**

다음 질문들은 산술 표현식을 작성하는 연습을 할 기회를 제공합니다.

1.  **42분 42초는 총 몇 초인가요?**  
    **정답:** `42 * 60 + 42 = 2562`초

2.  **10킬로미터는 몇 마일인가요? (힌트: 1마일은 1.61킬로미터입니다.)**  
    **정답:** `10 / 1.61 ≈ 6.21` 마일

3.  **10킬로미터 경주를 42분 42초에 완주했다면, 1마일당 평균 페이스는 몇 초인가요?**  
    **정답:**

      * 총 시간(초): `2562`초
      * 총 거리(마일): `6.21118` 마일
      * 1마일당 평균 페이스(초): `2562 / (10 / 1.61) ≈ 412.48`초

4.  **1마일당 평균 페이스를 분과 초로 나타내면 어떻게 되나요?**  
    **정답:**

      * 분: `412.48 // 60 = 6`분
      * 초: `412.48 % 60 ≈ 52.48`초
      * 따라서, 약 **6분 52초**입니다.

5.  **시속 몇 마일의 평균 속도인가요?**  
    **정답:**

      * 총 시간(시간): `2562 / 3600 ≈ 0.7117` 시간
      * 총 거리(마일): `6.21118` 마일
      * 평균 속도(mph): `(10 / 1.61) / (2562 / 3600) ≈ 8.73` mph

**모든 계산을 한번에 수행하는 코드:**

```python
# 문제 1
total_seconds = 42 * 60 + 42
print(f"1. 총 시간: {total_seconds}초")

# 문제 2
total_miles = 10 / 1.61
print(f"2. 총 거리: {total_miles:.2f}마일")

# 문제 3
seconds_per_mile = total_seconds / total_miles
print(f"3. 1마일당 평균 페이스: {seconds_per_mile:.2f}초")

# 문제 4
pace_minutes = int(seconds_per_mile // 60)
pace_seconds = seconds_per_mile % 60
print(f"4. 1마일당 평균 페이스: {pace_minutes}분 {pace_seconds:.2f}초")

# 문제 5
total_hours = total_seconds / 3600
miles_per_hour = total_miles / total_hours
print(f"5. 평균 속도: {miles_per_hour:.2f} mph")
```
